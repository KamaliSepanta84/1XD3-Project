2. Purpose

Our app, MacHub, is designed to help McMaster University students collaborate, share, and access high-quality class notes in an organized and rewarding way. By allowing students to securely upload their notes, categorized by course and topic. The app creates a hub of academic resources accessible to both peers and professors. To encourage academic collaboration and support, users can rate notes, leave feedback, and the highest-rated contributions will be recognized with rewards such as bonus marks or other incentives.

3. Data Description

  • User Data:
    • Each user is uniquely identified by their mac_id.
    • Users have attributes like name, email, and password (hashed for security).
    • A role field differentiates users as students, teaching assistants (TAs), or instructors.
    • The date_joined field records when the user registered.
  • Course Data:
    • Each course is uniquely identified by a course_code.
    • Courses have associated names, instructors, and semester details.
  • Notes Data:
    • Notes are uniquely identified by an auto-incrementing note_id.
    • Each note is uploaded by a user (linked via mac_id) and belongs to a specific course (course_code).
    • The title, description, and file_url store note details.
    • A timestamp records when the note was uploaded.
    • Notes can be public or private, managed through the visibility field.
  • Ratings and Reviews Data:
    • Ratings are uniquely identified by an auto-incrementing rating_id.
    • Each rating is linked to a specific note (note_id) and given by a user (mac_id).
    • Users can rate notes on a scale of 1 to 5 and leave comments.
    • A timestamp records when the rating was submitted.
  • Data Relationships:
    • Users upload notes, which are tied to specific courses.
    • Other users can rate and review these notes.
    • Deleting a user or course cascades the deletion of related notes and ratings, maintaining database integrity.

4. Functionality

  • User Registration/Login:
    • SELECT (to verify credentials during login)
    • INSERT (to register a new user)
    Client Side:
    • Form validation (e.g., email format, password strength)
    • Password hashing
  • Upload Notes:
    • INSERT into Notes table
    • File uploaded to storage
    Client Side:
    • File upload form with course selection dropdown Progress bar for upload status
    • Client-side file size/type validation
  • View Notes by Course or Rating:
    • SELECT from Notes table
    Client Side:
    • Note listing with search, filter, and sort functionality
  • Download:
    • SELECT note from Notes table
    • UPDATE download_count in Notes table
    Client Side:
    • Button to view PDF in-browser or trigger download
    • Update view/download count dynamically
  • Rate Notes:
    • INSERT into Ratings table
    • UPDATE average rating in Notes table (recalculated after each rating)
    Client Side:
    • Star rating input
  • Edit and Delete Your Own Uploaded Notes:
    • UPDATE Notes table (for edits to title/description)
    • DELETE from Notes table (and optionally delete file from storage)
    Client Side:
    • Edit/delete buttons shown only for user’s own notes
  • View Top Notes:
    • SELECT top-rated notes/users from Notes and Ratings tables
    Client Side:
    • Leaderboard UI with ranking, filters by course or time period
    • Badge/award icons for top users
  • Search For Notes:
    • SELECT from Notes table using keyword search (e.g., course code, title, description)
    Client Side:
    • Search bar
  • Receive Rewards (Admins Only Approve):
    • INSERT into Rewards table (by admin)
    Client Side:
    • Admin dashboard to approve/reject reward requests
    • User profile shows earned rewards and statuses

5. Role of PHP and JavaScript

In our app, PHP will be used primarily for server-side operations such as handling user registration and login, processing file uploads, managing database interactions (including inserting, updating, and deleting notes, ratings, and rewards). PHP will also serve as the backend for endpoints that JavaScript will communicate with. JavaScript, on the other hand, will be used for client-side functionality, including form validation, dynamic display of notes, search and sort features, real-time interactions such as rating notes, displaying leaderboards, and showing upload progress. JavaScript will use AJAX to send data to the PHP backend and update the page without requiring reloads. Together, PHP and JavaScript will work to create a smooth user experience with JavaScript capturing user actions and PHP securely processing them.

7. Incremental Delivery

The first increment of our app will focus on developing the frontend interface for uploading notes. This will include creating a clean and user-friendly upload page with a form that allows users to select a course from a dropdown menu, enter a title and description for their notes, and upload a file in PDF or DOCX format. Client-side validation will be implemented to ensure all fields are filled correctly and that only valid file types and sizes are accepted. A version of the login system may also be implemented, utilizing a mac ID and emails.
